# 書籍企画書

---

## 書名

『**LangChain**によるAIエージェント開発の教科書 〜自分だけのパーソナルアシスタント/汎用業務エージェント/コーディングエージェントを育てる〜』

---

## 概要

LangChain/LangGraph/LangSmithのエコシステムを体系的に習得しながら、パーソナルアシスタント/汎用業務エージェント（業務自動化エージェント）とコーディングエージェント（コード生成・修正・テストを行うAI開発支援ツール）をゼロから実装する「教科書」。

前著『LangChainとLangGraphによるRAG・AIエージェント[実践]入門』（技術評論社, 2024年11月）は基礎と設計パターンの入門書であったが、本書は「動くエージェントを作り、自分だけのものに育てる」ことに特化する。LangChain/LangGraphの基礎からDeep Agentsパターン、ツール設計、MCP対応、ローカルモデルの組み込み、システムプロンプト・UIのカスタマイズ、仕様ベースの継続的改善まで一冊でカバーする。

読後には、2種類のエージェントを構築・運用できるだけでなく、仕様を書いてエージェントの動作を定義・改善するサイクルを習得し、自社業務・開発環境に特化したエージェントに育てられる。

---

## 対象読者

- 任意のプログラミング言語での実務経験（TypeScript/Node.js推奨）があるソフトウェアエンジニア
- 前著読者（基礎を学んだ後、実際に動くエージェントを構築したい）
- 自社向けAIエージェントを設計・構築するエンジニアおよびAI担当者

前提知識: プログラミング経験。LangChainの事前知識は不要。

## 企画背景

GitHub Copilot、Claude Code、Cursor等のAIコーディングツールと業務自動化エージェントの普及により、AIエージェントを「使う」から「作って自社環境に合わせる」技術への需要が高まっている。

コーディングエージェントとパーソナルアシスタント/汎用業務エージェントの両方を一冊で体系的に扱い、ローカルモデルへの切り替えや業界特化型カスタマイズまで解説した日本語書籍は、著者調査の範囲では確認できていない。また、情報漏洩リスクへの対応としてクラウドLLMへの依存を避けたい企業（製造業・金融等）向けの実装書も不足している。

LangChain/LangGraph/LangSmithの基礎から実践的なエージェント構築・カスタマイズ・継続的改善まで一冊で体系的に学べる「教科書」として、前著読者の次のステップとなる書籍を企画する。

---

## 差別化ポイント

**1. 2種類のエージェントを一冊で実装**
パーソナルアシスタント/汎用業務エージェントとコーディングエージェントの両方を段階的に構築する。設計パターンの共通点と相違点を対比することで、エージェント設計の汎用的な知識が得られる。

**2. エージェンティックループとDeep Agentsによる実践設計**
固定的なワークフロー設計ではなく、タスクに応じて動的にフローが変化するエージェンティックループを中心に据える。LangGraph 1.0のStateGraph・Functional APIを実用に近い設計で解説するとともに、複数の推論・実行サイクルをまたいで複雑なタスクを自律遂行するDeep Agentsパターン（ディープリサーチ・セルフデバッグループ・長期タスクエージェント）を本書の核心的テーマとして扱う。

**3. ローカルモデルの利用やカスタマイズ実装までカバー**
Ollamaによるローカルモデルの運用、MCP対応ツールの設計、システムプロンプトとターミナルUIのカスタマイズを含む。特定クラウドプロバイダーに依存しない設計を一貫して採用する。

**4. 前著との発展的関係**
前著の読者にとっては「次に読む本」として自然な学習パスを提供する。前著未読でも本書単独で学べる構成とする。

**5. 仕様を書いてエージェントを「育てる」サイクルの習得**
エージェントの動作を仕様として定義し、ゴールデンセットによる評価とプロンプト改善を繰り返す継続的改善のサイクルを一章を割いて解説する。既製品を利用するだけでは得られない「自分だけのエージェントに育てる」プロセスを、LangSmithを活用して実践的に習得できる。

## 目次案

### 第I部 基礎（約110ページ）

**第1章 エージェンティックAIの全体像**（約25ページ）
- コーディングエージェントとパーソナルアシスタント/汎用業務エージェントの現在地
- エージェントの基礎概念: ReActパターン・ツール呼び出し・エージェントループ
- 本書で構築する2つのエージェントの全体像
- クイックスタート: 30行で動くエージェント

**第2章 LangChain 基礎**（約45ページ）
- LangChain 1.0のパッケージ体系と設計思想（langchain-core・langchain・langchain-{provider}）
- コラム：LCEL（LangChain Expression Language）: Runnableによるチェーン構成とパイプライン設計
- Message型とモデルの抽象化（AIMessage・HumanMessage・ToolMessage・SystemMessage）
- LangGraphのGraph API: StateGraph・Node・Edge・Conditional Edgeの設計
- コラム：LangGraphのFunctional API: @entrypoint / @task デコレータによるワークフロー定義
- LangGraphの高度な機能: Interrupt・Send API・Command・Subgraphによる複雑な制御フロー
- チェックポインティングと状態永続化（MemorySaver・SqliteSaver・PostgresSaver）
- LangSmithによるトレース・評価・デバッグ

**第3章 エージェンティックループとDeep Agentsパターン**（約40ページ）
- ワークフローとエージェンティックループの違いとトレードオフ
- 基本設計パターン: ReAct・Plan-and-Execute・Reflexion
- Deep Agentsパターン: 長期・多段階タスクを自律実行するエージェント設計
  - ディープリサーチエージェント: 検索→収集→分析→レポート生成の多段階ループ
  - セルフデバッグループ: 実行失敗を自己分析して修正するサイクル
  - 長期タスクエージェント: チェックポインティングを活用した中断・再開設計
- Human-in-the-Loopパターン（interrupt/resume機構）
- ハーネスエンジニアリング入門: エージェントが失敗する原因と環境設計の考え方
- エラーハンドリングと停止条件の設計

---

### 第II部 コア技術（約125ページ）

**第4章 ツール設計とMCP**（約35ページ）
- ツール設計の原則（単一責任・明確なスキーマ定義）
- ファイル操作ツール群とシェル実行ツール
- MCP（Model Context Protocol）: 2024年にAnthropicが発表し2025年に主要ベンダーが採用した業界標準プロトコル
- MCPサーバーとしてのツール公開・クライアントとしての利用
- サブエージェントとスキルの設計

**第5章 プロンプト設計とシステムプロンプト**（約30ページ）
- システムプロンプトの構成要素（ロール定義・制約条件・出力フォーマット）
- コーディングエージェント向けプロンプト設計（コード生成・エラーフィードバック）
- パーソナルアシスタント/汎用業務エージェント向けプロンプト設計（タスク管理・確認フロー）
- Few-Shotパターンとプロンプトキャッシング

**第6章 コンテキスト管理とメモリ**（約25ページ）
- 短期メモリ（会話履歴・ツール実行結果のキャッシュ）
- 長期メモリ（知識・ユーザー設定の永続化・ベクトルストア）
- コンテキストウィンドウ管理（圧縮・優先度付け）

**第7章 エージェントの評価（Evaluations）**（約35ページ）
- 評価の3層: ユニット評価（ツール単体）・タスク評価（1タスク完了）・業務評価（ビジネス指標）
- ベンチマーク評価: SWE-bench・GAIA等の標準ベンチマークと業務固有データセットの設計
- LLM-as-Judgeパターン: 評価基準の定義・ゴールデンセット・バイアス制御
- 業務評価とインサイト取得: プロダクション環境のトレースからKPI・失敗パターンを分析
- 評価→インサイト→プロンプト/ツール改善のサイクル（LangSmith評価パイプライン）
