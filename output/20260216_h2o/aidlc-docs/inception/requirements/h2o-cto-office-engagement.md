# H2Oリテーリング × Section-9 「CTO室」型エンゲージメント構想

**作成日**: 2026-02-16
**作成方法**: AI-DLC INCEPTION Phase（Requirements Analysis）
**情報ソース**: H2O戦略ブリーフィング（2026-02-13）、Backlog C2GAAI実績、GAサービスカタログ
**目的**: 小山社長とのディスカッションに向け、sec9チームがCTO室のような立ち位置で継続的に関与するエンゲージメントモデルを提案する

---

## エグゼクティブサマリー

H2Oリテーリングの開発本部は、約2年にわたるSection-9との協業を通じて「御用聞きコンサルから戦略的パートナーシップへの移行」に合意した（2026/1/28定例）。しかし現状は6つのワークストリームが個別に進行し、横断的な技術ガバナンスが不在の状態である。

本構想では、Section-9が**仮想CTO室**として機能し、H2O開発本部の技術意思決定・ベンダーマネジメント・アーキテクチャガバナンスを一元的に支援するモデルを提案する。

---

## 1. 背景：なぜ「CTO室」なのか

### H2O開発本部の現状課題（FigJamワークショップ 2/4より）

| 課題 | 具体例 | CTO室で解決できること |
|------|--------|---------------------|
| ベンダー提案の選定困難 | Vinx / SCSK / ecbeing / Server Works等の技術評価 | 技術評価基準の標準化、RFPプロセスの確立 |
| 組織の役割分担と評価 | 開発本部 vs 事業会社の役割不明確 | 技術組織設計、スキルマトリクス策定 |
| ガバナンスとスピードのバランス | 承認プロセスの遅延 vs 技術リスク | アーキテクチャレビューボードの運営 |
| 開発コストの高騰 | AWS費用、ベンダー費用の増大 | FinOps・技術投資ポートフォリオ管理 |
| 技術的意思決定の属人化 | 小山さんに判断が集中 | 技術意思決定フレームワークの確立 |

### Section-9が適任である理由

1. **2年間の文脈蓄積**: H2Oの技術課題・組織課題・ベンダー関係を深く理解
2. **横断的な技術力**: AWS、セキュリティ、性能最適化、AI活用を一社でカバー
3. **AI-DLC方法論**: 意思決定の構造化・文書化を自動化し、属人化を解消
4. **KHI等での実績**: Backlog C2GAAIプロジェクトで技術アドバイザリー + プロトタイピングの実績

### 7つの機能ドメイン

---

### 2.1 技術戦略策定・ロードマップ管理

**概要**: H2O全体の技術戦略を策定し、四半期ごとにロードマップをレビュー・更新する

**具体的な活動**:
- 年間技術ロードマップの策定（九州統合2027/7を見据えたマイルストーン設計）
- 四半期レビュー会議（小山社長 + 開発本部 + Section-9）
- 技術投資ポートフォリオの管理（攻め：EC基盤 / 守り：BCP・セキュリティ のバランス）
- 業界トレンド・新技術の定期レポーティング

**H2Oへの価値**:
- 技術意思決定の「見える化」と「脱属人化」
- 九州統合に向けた技術準備の確実な進行管理
- 来期予算配分の根拠となるデータ提供

**GAサービスとの対応**: アドバイザリー Plan B（月4回）+ カスタムロードマップ策定

---

### 2.2 ベンダーマネジメント・RFP支援

**概要**: 複数ベンダー（Vinx, SCSK, ecbeing, Server Works等）の技術評価・管理を標準化する

**具体的な活動**:
- **RFP標準テンプレートの策定**: 技術要件・評価基準・SLAを統一
- **ベンダー技術評価レポート**: 各ベンダーの技術提案を独立した視点で評価
- **SLAモニタリング**: Vinxのような遅延問題の早期検知と是正アクション
- **マルチベンダーガバナンス**: ベンダー間の技術整合性を確保

**H2Oへの価値**:
- BCP案件でServer Works vs Section-9を客観評価できる枠組み
- Vinx遅延問題の構造的解決（SLAベース管理への移行）
- ベンダー選定の「勘」から「基準」への移行

**具体例（IT BCP）**:
- Section-9が設計書レベルのRFP回答を作成（Server Worksとの差別化）
- 同時に、ベンダー評価フレームワークとしてH2O側にRFP評価プロセスを残す

---

### 2.3 アーキテクチャレビューボード（ARB）

**概要**: 全プロジェクトの技術設計を横断的にレビューし、品質とガバナンスを確保する

**具体的な活動**:
- **月次ARBミーティング**: 新規・変更案件の技術設計レビュー
- **アーキテクチャ原則の策定**: H2O共通の技術標準（命名規則、セキュリティ基準、API設計等）
- **技術的負債の管理**: 定期的な技術負債の棚卸しと返済計画策定
- **クロスプロジェクト技術整合**: BCP / MDWare / EC基盤間の技術的依存関係管理

**H2Oへの価値**:
- MDWare性能改善とBCPの設計整合性を事前に確保
- ecbeing導入時のアーキテクチャ標準適合チェック
- 技術的負債の可視化と計画的な返済

**具体例（MDWare性能改善）**:
- Vinxの改善提案をARBでレビューし、技術的妥当性を検証
- インスタンス2倍化（¥2,000万） vs クエリ最適化の技術的根拠を整理

---

### 2.4 AI/DX推進オフィス

**概要**: AI活用戦略の策定と実行を推進し、組織全体のDXを加速する

**具体的な活動**:
- **AI活用戦略の策定・実行**: 30日実験（小西担当）の結果を本格ロードマップに展開
- **「AI竹内さん」構想の実現**: 経営判断支援AIシステムの設計・構築
- **AI-DLC全社展開**: 見積もり・設計・レポート作成の自動化を全プロジェクトに横展開
- **Claude Code導入支援**: 開発本部メンバーへのAIコーディング実践講座
- **AI Agent活用のPoC**: AWS AI Agent（Security Agent / DevOps Agent）の検証

**H2Oへの価値**:
- 経営判断スピードの向上（情報格差の解消）
- 開発プロセスの効率化（AI-DLCによる設計自動化）
- 開発本部メンバーのスキルアップ

**GAサービスとの対応**:
- 研修プログラム: AIコーディング実践講座（Level 2）
- エージェント導入支援: 導入設計 + プロトタイピング
- アドバイザリー Plan C: プロトタイピング付き技術支援

---

### 2.5 セキュリティ・ガバナンスオフィス

**概要**: 全社的なセキュリティ体制の構築と継続的な脆弱性管理を行う

**具体的な活動**:
- **脆弱性トリアージの自動化**: Security Hub 1,000件超の脆弱性をAIで自動分類・優先度付け
- **DevSecOpsパイプライン標準化**: GitHubレビュー自動化を全プロジェクトに展開
- **CyberArk/Tenable統合管理**: 特権管理とスキャン管理の一元化支援
- **ISMS認証取得支援**: 開発本部スコープでの認証取得プロセス設計
- **セキュリティBCP**: インシデント対応手順の策定と訓練

**H2Oへの価値**:
- 脆弱性対応工数の大幅削減（AIトリアージ）
- セキュリティインシデントリスクの低減
- ISMS認証取得による対外信頼性向上

---

### 2.6 FinOps（クラウドコスト最適化）

**概要**: AWS利用費の継続的なモニタリングと最適化を行う

**具体的な活動**:
- **月次コストレビュー**: AWS利用費の推移分析と異常検知
- **即効施策の実行**:
  - S3: Google Security Operationsスキャン設定最適化（年間¥3,600万〜4,800万削減）
  - FSX: ストレージ階層の最適化（月額¥250万の20-30%削減）
  - EC2/RDS: Savings Plan/RI戦略の策定と実行
- **予算計画支援**: 来期AWS予算の策定根拠データ提供
- **FinOpsダッシュボード構築**: リアルタイムのコスト可視化

**H2Oへの価値**:
- 年間数千万円規模のコスト削減（即効性あり）
- 予算管理の精度向上
- コスト異常の早期検知

---

### 2.7 技術人材育成・組織設計

**概要**: 開発本部の自走化に向けた人材育成と組織設計を支援する

**具体的な活動**:
- **スキルマトリクスの策定**: 開発本部メンバーの現状スキルと目標スキルの可視化
- **段階的自走化プログラム（3-6ヶ月）**:
  - Month 1-2: コーチング（Section-9が主導、H2Oが学ぶ）
  - Month 3-4: 協働（共同で意思決定・実行）
  - Month 5-6: 自走（H2Oが主導、Section-9がレビュー）
- **組織再編支援**: 開発本部を戦略・標準策定に特化させる組織設計
- **評価制度設計**: 技術力と成果の両面を評価する仕組み

**H2Oへの価値**:
- 組織の自走化実現（2年間の課題解決）
- 人材の定着と成長
- 開発本部の役割明確化

**GAサービスとの対応**:
- 研修プログラム: 生成AI基礎講座（Level 1）→ AIコーディング実践（Level 2）
- BYNAME: 必要に応じたAIエージェント人材の紹介

## 5. 小山さんとのディスカッションポイント

### 問いかけ案

1. **「開発本部に必要なのは、外部CTO室ではないでしょうか？」**
   - 2年間の協業で蓄積した文脈を活かし、技術意思決定を構造化
   - 小山さんへの判断集中を解消し、組織的な意思決定へ移行

2. **「まずはクイックウィンで成果を示し、段階的に機能を拡大しませんか？」**
   - S3コスト削減（年間¥3,600万〜）を即座に実行し、CTO室の価値を証明
   - Phase 1の成果をもとに、Phase 2以降の投資判断を行う

3. **「九州統合（2027/7）のカウントダウンは始まっています」**
   - データ量1.5倍への対応は今から着手しないと間に合わない
   - CTO室がロードマップを管理し、確実に進行させる

4. **「御用聞きから脱却した新しい関係を、形にしませんか？」**
   - 1/28合意の「戦略的パートナーシップ」を具体的な組織モデルとして定義
   - CTO室という明確な役割定義により、双方の期待値を合わせる

### 想定される懸念と回答

| 懸念 | 回答 |
|------|------|
| コストが高い | Phase 1はPlan A（¥800K/月）から開始可能。ROI 2〜4倍の試算あり |
| 社内に定着するのか | 自走化プログラム（3-6ヶ月）を組み込み済み。Section-9の撤退基準を明確化 |
| 既存ベンダーとの関係は | CTO室はベンダーと競合しない。むしろベンダー管理を改善する立場 |
| 他社でも同じことでは | 2年間の文脈蓄積は代替不可能。AI-DLCという独自方法論がある |
