## 2. コード例のPython構文エラー

### 検出された問題

#### 問題1: `dict[str, any]` — 型アノテーションの `any` が小文字（重要度: 高）

**ファイル**: chapter-06.md
**行**: 1742, 1766

```python
# 誤り（現状）
def run_tests(
    self, test_path: Optional[str] = None
) -> dict[str, any]:  # ← 'any' は小文字

def format_result(self, result: dict[str, any]) -> str:  # ← 同上
```

**問題点**: Pythonの型システムにおいて `any` は組み込み型ではない。正しくは `typing.Any`（大文字始まり）を使用する必要がある。小文字の `any` はPythonの組み込み関数 `any()` を指し、型アノテーションとしては意味が異なる。実行時にはエラーにならない（Python 3.10以降の `dict[str, ...]` 構文ではアノテーションは評価されないため）が、型チェッカー（mypy, pyright等）ではエラーとなる。

**修正案**:
```python
) -> dict[str, Any]:  # typing.Any を使用
```

**同ファイル内の他のコード例ではすべて `dict[str, Any]` （大文字）が正しく使用されている**ことから、これはタイポと判断される。章内で計20箇所以上 `dict[str, Any]` が正しく使用されている中、この2箇所のみが小文字となっている。
