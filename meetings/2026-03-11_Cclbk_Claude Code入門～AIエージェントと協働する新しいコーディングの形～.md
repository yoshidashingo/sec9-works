#### Agentic Codingの流れ
* AIコーディングは第**1**世代（コード補完）→第**2**世代（チャットでメソッド・クラスレベルの支援）→第**3**世代（自律的に開発を進めるエージェント）と進化してきた
* 第**3**世代は**2024**〜**2025**年にかけてClaude CodeやDevinなどが登場し、「人がAIに依頼する」から「AIがリードして人間が時々手伝う」モデルへシフトしている
* Claude Codeは**2025年2月**にリサーチプレビューとして公開（当初はClaude 3.5）、**5月**にClaude 4.0 Sonnet/OpusのリリースとともにGAになった
* AIエージェントの本質は「環境認識→行動計画→行動→結果知覚」のループで、LLMが中心である必要はない
* Claude Codeはそのエージェントハーネス（LLMに手足となるツール群・メモリ・ポリシーを組み合わせた仕組み）にあたる
* エージェンティックワークフロー（DifyやN8Nなど事前定義型）と異なり、エージェントハーネスはユーザーの要求に応じてワークフローをオンデマンドに生成できる点が大きな違い

#### 仕様駆動開発デモ
* CLAUDE.mdというメモリファイルに開発ルールを定義しておくと、Claude Codeが起動時に必ず読み込んでルールに従って動作する
* デモでは開発標準ルールに基づいてプロダクト企画書・要求仕様書・機能設計書・技術仕様書・アプリ構造定義書を自動生成し、最終的にタスクリストまで出力
* 仕様駆動開発の基本は「これらの仕様書を唯一の信頼できる情報源（Single Source of Truth）として、ソースコード開発を進める」こと
* 詳細な実践は**3月25日**のVol.2で扱う予定

#### AIDLCとは
* AWSが提唱するソフトウェア開発ライフサイクルのフレームワークで、定義・リファレンス実装ともにGitHubで公開されている
* Inception（要件定義）・Construction（開発）・Operation（運用）の**3**フェーズで構成され、AIがリードして人間が承認ゲートとして意図（Intent）を注入していく
* 最大の特徴はMob Elaboration（関係者が一箇所に集まってAIのスピードに合わせて詳細化していく）という考え方
* 吉田は全ての仕事においてAIDLCワークフローを参照ルールの一部として採用しており、特にInceptionフェーズの出来が作業品質の大部分を左右すると考えている
* AIDLCを骨抜きにせず体得することが重要で、まだAWSでも対応できる人材が少なく情報も出回っていないのが現状
* 詳細な実践は**3月25日**のVol.2で扱う予定

#### 業務活用とMCPデモ
* **2025年5月**のClaude Code Actions登場以降、コーディング以外のあらゆる業務（商談準備・議事録まとめ・レポート作成・分析など）にも活用できると認識が広まった
* MCPを使うとBacklog・Gmail・Google Drive・Google カレンダーなどの外部サービスとClaude Codeを連携できる
* MCPの設定場所は**3**種類あり、プロジェクト固有ならプロジェクトのMCP.json、常時使いたいならホームのClaude.json、Web・コワーク両方で使いたいなら組み込み連携を利用する
* デモではBacklogのWikiにある議事録を読み取り、Google Driveのテンプレートを使って顧客ごとの営業フォローアップ資料をGoogle スライドで自動生成
* Claude.appデスクトップのCore Arcのスケジュールタスク機能を使うと、早朝・深夜に定時タスクとして自動実行できる（例：Google カレンダーから翌日の商談を確認して準備資料を事前作成）
* 吉田は議事録収集・フォローアップ資料作成・商談準備・定例アジェンダ出し・Backlogの未解決タスクへのコメント生成などをスケジュールタスクで自動化している

#### Claude Codeの課題とTaskZero
* 現状のClaude Codeはローカル実行のため、PCがシャットダウンするとタスクが止まるという制約がある
* AIと個々のユーザーが**1対1**でしか仕事できず、チームでのMob Elaborationや上位者の介入・コンテキスト補完ができない
* これらの課題を解決するためにTaskZeroを開発中で、クラウド上での実行環境・チームコラボレーション・外部データテーブル・定期起動などをClaude Codeの上位ハーネスとして提供する

#### Q&A
* **AWSセキュリティ情報の管理**: AWS CLI経由でクレデンシャルを管理するため、エージェント専用の特別な設定は不要
* **AIDLCの初心者向けリソース**: 専用の入門サイトはまだほぼなく、公開されている情報を自分で読んで実践するのが現状のベストな方法
* **AIDLCのカスタマイズ**: AIDLC自体はいじらず、Extensionフォルダにセキュリティベースライン・サービスデザイン・テクニカルライティング・メールの書き方・チェンジマネジメントのルールを追加して共存させている
* **CLAUDE.mdの作成**: リポジトリ内でClaude Codeを実行すると自動生成されるが、単なるMarkdownファイルなので自由に作成・編集・削除して構わない
* **セキュリティ面**: DevContainerの中でローカル実行するのが最も安全で、Fintanの記事と書籍に詳細あり
* **AIコーディングで必要なマインドチェンジ**:
1. 仕組みづくりに優先的に時間を割く（1日1〜2時間）— 斧を研がないと木を切る時間が余計にかかる
2. 常識を疑うクリティカルシンキング — 人間によるレビューは物量的に破綻することが見えているので、AIにレビューさせて人間は例外ケースだけ確認する仕組みに変えていく必要がある
* **SaaSの未来**: 短期的には大きな問題ではないが、MCPを使えばエンタープライズ機能相当のことを自前で実現できるようになるため、SaaSのアップセル機会が潰される可能性はある
* **Claude Code vs 競合（Gemini/ChatGPT等）**: スイッチングコストが低いので、気になるなら実際に課金して自分で評価してサブスクを残すものを決めるのが一番確実
