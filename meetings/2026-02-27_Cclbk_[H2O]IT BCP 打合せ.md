#### Overview
* FSxのクロスリージョンコピーは別オーガニゼーション間で非対応のため、S3 Data Syncを使った代替方式を採用
* 切り戻し方針を明確化：大阪リージョンでバックアップ取得は実施するが、レプリケーションとアプリケーション設定は手動対応
* プロジェクトスコープを整理：Lambda Python EOL対応とTGW構築は対象外、大阪リージョンのランニングコストは参考情報として提示
* コスト配分は設計フェーズ**880万円**、構築フェーズ**1,100万円**で調整中
* 提案書は要件定義書、コスト見積もり、依頼事項のマークダウンをPDF化して提出予定


#### 要件定義書の確認とDR対策の全体像
* 対策の前提脅威はランサムウェア攻撃で、店舗PCが攻撃を受けてシステム停止した状態からの復旧を想定
* IT側の対応範囲はシステム再構築とデータ復旧で、インフラ復旧はH2Oのマルチクラウド運用チームが担当
* 復旧方法は東京リージョンから大阪リージョンにバックアップデータを複製し、大阪リージョンで復旧
* 目標復旧地点(RPO)は**90日**以内のいずれかの時点
* 目標復旧時間(RTO)はKFMが**2〜3日**、百貨店は未定義のため**4月**以降に検討
* 復旧プロセスは**7週目**から開始する前提でロードマップを作成
* 原因調査や侵入経路の特定は今回のスコープに含まないことを明記


#### 切り戻し方針とバックアップ体制
* 大阪リージョン稼働後の切り戻しについて、バックアップは大阪側で取得するが緊急性は低い(約**1年後**を想定)
* 切り戻し時のレプリケーションとアプリケーション設定は手動対応とし、別途訓練や手順書の検討が必要
* 大阪リージョン稼働後は大阪から東京へ逆向きのバックアップ体制を構築
* **42枚目**のスライドに記載：大阪リージョンにバックアップを保管しつつ、東京リージョンに複製を作成する方針


#### FSxクロスリージョンコピーの技術的課題
* FSx for Windows File Serverは別オーガニゼーション間でのクロスリージョンコピーが非対応
* AMIやRDSスナップショットは別組織間でイメージシェアが可能だが、FSxは同一オーガニゼーション前提
* 代替案としてData Syncを使ってS3にエクスポートし、S3クロスリージョンレプリケーションで大阪リージョンに転送
* S3のバージョニングを有効化することで特定時点へのポイントインタイムリカバリー(PITR)を実現
* データ同期には**5〜9時間**かかる見込みで、容量も大きくなる
* 月次コストは約**3,000ドル**程度と試算
* EC2とRDSのバックアップはAWS Backupを使用し、ステップファンクションズで再暗号化の仕組みを構築


#### プロジェクトスコープの明確化
* Lambda Python EOL対応はアプリベンダーのスコープとし、対象関数リストの提示のみ実施
* TGW(Transit Gateway)の作成は**3月5日**にGICTグループが対応予定のため対象外
* 百貨店POSのRTO未定義については、要件定義段階のため**4月**以降に検討し詳細を記載
* サーバー台数は要件定義書で**27台**となっているが実際は**18台**の可能性があり、現況調査後に対応表を追加
* RDSがサーバー台数に含まれていない可能性があり、EC2とRDSの内訳を明確化
* ステップファンクションズはバックアップ系のみが対象で、ApplyJobやApplyVendorのスコープに含まれる
* 大阪リージョンのランニングコスト試算はスコープ外とし、Appendixに参考情報として記載


#### コスト見積もりと工数配分
* 基盤設計フェーズは**640人日**で見積もり中
* 全体コストは設計・計画フェーズが**880万円**、構築フェーズが**1,100万円**の配分
* 構築フェーズの**1,100万円**はServerWorksに委託する可能性がある
* H2O側の想定ではServerWorksの見積もりが**1,500万〜1,800万円**程度
* Section-9の提案は**20%**のコスト削減を実現
* 基盤テストと構築フェーズの工数配分を再調整中で、テスト計画の**12日**は不足している可能性


#### 提案書の最終化
* 提案書はプロポーザルをアップデートする形で作成
* 提出物は要件定義書、コスト見積もり、依頼事項のマークダウンをPDF化して提出
* 要件定義書の内容をClaudeに再度読み込ませて矛盾がないか確認が必要
* FSx Data Syncの詳細な方式とリスクを提案書に明記
