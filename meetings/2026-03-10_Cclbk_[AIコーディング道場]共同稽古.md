#### API キーのセキュアな管理方法
* Keita は `.env` に本番 API キーを置くとプロンプトインジェクションで外部に漏洩しうるリスクを認識しており、Gemini API キーと GCP 認証情報の安全な管理方法を相談した

* habuchin は自分が使っている方法として以下を紹介した：
* GitHub Actions の `env secrets` / グローバルシークレットにキーを格納し、デプロイ時のみ使用する
* 1Password の `.env` 管理機能でローカルの `.env` を 1Password と同期させる
* `op` CLI（1Password CLI）で環境変数をエクスポートし、標準出力に出さずに取り込む
* AWS Secrets Manager や GitHub CLI (`gh` コマンド) でシークレットを取得する方法もある


#### LLM 入出力の設計とリスク対策
* habuchin は Gemini の無料プランではユーザーデータが学習に使われる利用規約になっているため、ユーザーのメールアドレス等を LLM に渡す可能性があるなら有料プランへの切り替えを検討すべきだと指摘した
* LLM を API で使う際のベストプラクティスとして habuchin が共有した内容：
* LLM に渡す用途を限定する（スコープを絞る）
* 出力を JSON 形式に固定し、それ以外を出力させないようにする
* この **2** 点を組み合わせることで、プロンプトインジェクションによる SQL インジェクション等への波及を防げる
* 設計時に AI に相談するとベストプラクティスを出してくれるため、Claude に相談しながら進めることを habuchin は勧めた
