#### 概要
* ストーリーボードのフロー（ステップ1〜4）を確認し、まずは**1**ユニット単体での実装を進めることに決定
* OpenAI キーの問題で同じ画像が生成され続けているため、真吾が新しいキーを払い出す
* Sora API が使えないため、動画生成を Gemini に切り替える方針に決定
* ステップ間の「戻る」動作の仕様を確認・合意

#### ストーリーボードのフロー確認
* フローはラフアップロード → 画像生成 → 選択 → 動画生成 → 修正の順で構成
* 複数ラフをアップロードすれば複数ストーリーを生成できる設計だが、まずは**1**ユニットで進め、複数ユニット対応は後回しにすることに決定

#### OpenAI キーの問題
* 現状、同じ画像が繰り返し生成されており、OpenAI キーに起因している可能性が高い
* 真吾が新しい OpenAI API キーを払い出して共有する

#### Gemini への動画生成切り替え
* Sora API が利用できないため、動画生成を Gemini に切り替えることに決定
* 湯は動画生成において Gemini が現状最も優れており、字幕・BGM も付いてくると評価
* 現状の生成動画には字幕・BGM がついていないという課題があり、Gemini への切り替えで解消できる見込み

#### ステップ間の戻る動作の仕様
* 戻ってプロンプトを変更せずに「次へ」を押した場合は再生成せず、単純なページ移動のみ
* プロンプトを変更して「次へ」を押した場合は再生成が走り、それまでに生成したコンテンツはすべて上書き・消去される
* ステップ7まで進んだ後に修正したい場合は、戻るよりも新しいストーリーとして作り直す方が望ましいという方向性を確認（大きな問題ではなく、引き続き検討）
