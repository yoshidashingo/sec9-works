#### 概要
* MSAI Tourは既存顧客中心の集客でデベロッパーエンゲージメントがなく、PingCAPブースへの反応も薄かった
* GitHub Copilotはプルリク駆動開発のような方法論の提示がなく、「VSCodeから使う以上のもの」がないため哲学が透けて見えてしまっているとYoshidaは見ている
* PingCAP社内ではCTOとVP主導でAI活用がオーガニックに広がっており、サポートエージェントのスキル開発モデルが面白い事例になりそう
* 次回MTGは**4月10日**同時刻、Findy登壇コンテンツのレビューとTiDB User Day企画のフィードバックが議題

#### MSAI Tourの振り返り
* ブースに来た人のほとんどがAIに馴染みのない既存MSユーザー企業で、コパイロットも「使っちゃダメ」という企業がちらほらいた
* イベント自体がCopilot StudioやFabric推しで、エージェント作成ソリューションの紹介に終始しており具体的なユースケースが乏しかった
* ブース専用Wi-Fiがなく、JBCCのNinaさんも「デモができなかった」と話していた
* ディスプレイが**24**インチ程度で会場が広い分スカスカな印象で、目の前に来てもらわないと何も見えない状態だった
* YoshidaはMSが新規チャンネル開拓をせず既存顧客チャンネルに頼り続けていることが根本原因と見ている
* AOAIブースもPingCAPと同程度の集客で、開発者向けイベントとしての性格が薄かった
* 今後の出展では「AI駆動開発」か「パーソナルエージェントでこれとこれを自動化できました」レベルの超具体パネルでないと反応が取れないとYoshidaは考えている

#### GitHub Copilotのエンゲージメント不足
* GitHubブースはステッカー配布とCopilotで作ったゲームのデモのみで、Copilot自体の説明やセミナーがなかった
* YoshidaはCopilotが売れていた理由をプルリク駆動開発などの方法論とセットで訴求していたからだと分析している
* 今はinstruction.mdを書いてくださいと言うだけで「日々の開発がどういうスタイルであるべきか」を誰も提示していないため、デベロッパーに哲学のなさが透けて見えてしまっている
* Yoshidaのクライアントの約半数がCopilotしか使えない環境にあるが、実際には使っていない

#### PingCAP社内AI活用の実態
* CTOがDB9を作り始め、VPは自分のPCにOpenClineを入れてリモート開発環境の修復などに使っており、トップ2人がAIにどっぷりはまっている
* VPはOpenClineでMemlineを作り始め、それがサブプロダクトからメインプロダクトに昇格していった
* サポートチームが「サポートを全部AIでできるんじゃないか」というコンセプトでOpenClineのハーネスを構築し、Jiraやメトリクス読み込みから始まりログのABCPやパフォーマンスデータ抽出までスキルを肉付けして今は「一人立ちできる」状態になっている
* 普及の流れはコアチーム（二重系・三重系）が先行してスキルを育て、フロントライン（一重系）に展開するという順序で進んでいる
* 各AIエージェントプロダクトごとにオープンチャットグループがあり、コアチームへの機能要望やフィードバックが飛び交っている
* 会話をシェアする機能がエージェントに最初から入っており、「このスキル足りないね」「このスキルよくないね」という議論がその場でできる点をSekiguchiは評価している
* 全体としてまだ関係部門へのヒアリングはしておらず、社内の動きを観察している段階

#### AI浸透度の測り方
* Yoshidaは社員一人ひとりのAI浸透度を数値で出せないか提案した
* トークン数よりも、エージェントやチャットで発生するタスク数やターン数で見る方が実態を反映しやすいとYoshidaは考えている
* PingCAPではClaude APIやOpenAI APIの費用を集中部門が予算を持って一括負担しており、社員は自由に使える環境になっている

#### OpenClineのコンテキスト管理
* OpenClineのコンテキスト管理が悪く、直前の会話を全部・関係あるものもないものも含めて差し込みコンテキストに大量に入れてくる
* 外部のClaude CodeやCodexモデルを使うとトークン消費が激しく課金が耐えられないレベルになる
* Memlineのセットアップ時にDiscordから「これを覚えておいて」系の書き込みオペレーションが使えない問題があり、OpenCline側がデフォルトでチャットチャンネルへの書き込み系オペレーションを非公開にしているのが原因
* OpenClineのエコシステムは他のクローンと比べて圧倒的に巨大で開発も活発だが、誰も全容を把握できていない状態
* 記憶の圧縮はホットトピックになっているが、Yoshidaはまだ人類に解けていない問題だと見ている
