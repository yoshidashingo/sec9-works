#### AIDLC GitHub App デモ
* 宏樹が GitHub Issue にラベルを付けると AIDLC ワークフローが起動する仕組みを作り込んだ
* ラベル付与 → GitHub App が受け取り → Webhook で API Gateway を叩く → バックエンドは API Gateway + Lambda 構成
* 要件が不明確な箇所を AI が質問で収束させるラウンドを繰り返し、仕様書を自動生成する
* 品質ゲートを各ステップに組み込み、スコアが合格になるまでサブエージェントが自動リトライする
* 品質ゲートの観点・プラグインエージェントの構成は YAML ファイルで全て指定できる設計
* 会話ログ・監査ログは DynamoDB に記録、CloudWatch Logs にも出力するため後から参照可能
* AIDLC のアーティファクトをリポジトリ内に置く必要がなく、Issue 上で全て追跡できる点を真吾が評価
* 自前の実行基盤（AWS 上）を持つほうが、GitHub Actions だけで回すより柔軟でフック・ワークフローの複雑さにも対応しやすいという結論
* AWS SA の金森さんに見せたところ「ここまでやっているのはグローバルでもなかなかない」と評価、ヨーロッパは **2月**頃、米国は **12月**頃から始めたばかりで日本が先行している
